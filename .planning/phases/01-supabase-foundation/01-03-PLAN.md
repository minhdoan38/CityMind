<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser to Supabase Auth | Untrusted credentials create an SSR cookie session. |
| Next.js SSR/BFF to FastAPI | Access tokens cross a server-to-server boundary for officer requests. |
| Public routes to dashboard routes | Route authorization must prevent unauthenticated or wrong-role rendering. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-09 | Spoofing | SSR session authorization | high | mitigate | Use `getClaims()` for authorization, refresh cookies in proxy, and fail closed on missing/wrong roles. |
| T-01-10 | Information Disclosure | access token forwarding | high | mitigate | Keep tokens in HttpOnly cookies/server code, forward only Authorization bearer headers, and never log them. |
| T-01-11 | Elevation of Privilege | dashboard/BFF routes | critical | mitigate | Apply officer/admin guard at route/layout and BFF boundaries; FastAPI independently verifies the same JWT. |
| T-01-12 | Repudiation | officer actions | medium | mitigate | Preserve authenticated actor principal through status updates for append-only event attribution where schema supports it. |
| T-01-SC | Tampering | npm installs | high | mitigate | No new installs occur here; consume exact audited pins and the single approval recorded by Plan 01. |
</threat_model>
