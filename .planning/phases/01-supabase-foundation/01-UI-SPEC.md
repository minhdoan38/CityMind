## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- **44px** minimum touch height/width for primary public CTAs and login primary button (WCAG target size; between xl and 2xl)
- Sidebar collapsed icon rail may use **48px** row height (2xl) — already on scale

---

## Typography

Exactly 4 sizes, 2 weights (400 + 600):

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Label | 14px | 400 | 1.4 | Form labels, nav items, helper text, status chips |
| Body | 16px | 400 | 1.5 | Instructions, login copy, placeholder body |
| Heading | 20px | 600 | 1.2 | Page titles (Login, Dashboard), section headers |
| Display | 28px | 600 | 1.2 | Public shell brand lockup / placeholder hero title only |

**Rules:** No third weight. No fluid clamp in dashboard chrome. Public display may use `clamp(1.5rem, 4vw, 1.75rem)` capped at 28px. Letter-spacing on display ≥ -0.04em. Body prose max ~70ch.

---

## Color

60 / 30 / 10 split (restrained civic):

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` | Page background, main content canvas |
| Secondary (30%) | `#F0F4F3` | Sidebar, header bar, muted panels, login card surface |
| Accent (10%) | `#0F766E` | See reserved list below |
| Destructive | `#B42318` | Auth error emphasis / future destructive only — not primary logout |
| Ink | `#14201D` | Body and headings (≥7:1 on white) |
| Muted text | `#5C6B66` | Secondary labels (≥4.5:1 on white) |
| Quiet line | `#D5DEDB` | Borders, separators |

Accent reserved for (only):
1. Primary CTA fill (Officer sign-in; public “Report an issue” placeholder link/button)
2. Active sidebar nav indicator / selected item
3. `:focus-visible` rings on interactive controls
4. Text links that are primary actions (not every `<a>`)

**Not accent:** default borders, icons at rest, table chrome, decorative backgrounds, hover washes (use secondary/ink opacity instead).

**Forbidden this phase:** purple gradients, neon glow, dark cyber shells, cream/sand body backgrounds.

## Copywriting Contract

EN defaults (VI strings mirror meaning; add in i18n scaffold):

| Element | Copy |
|---------|------|
| Primary CTA (login) | Sign in to dashboard |
| Primary CTA (public shell) | Report an issue |
| Login heading | Officer sign-in |
| Login supporting | Use your CityMind officer account. AI suggestions stay advisory — you decide. |
| Empty state heading | No reports to show yet |
| Empty state body | When citizens submit issues, they will appear here. Check filters or refresh after the first report lands. |
| Error state (auth) | Could not sign in. Check email and password, then try again. If this keeps happening, contact your admin. |
| Error state (shell load) | Could not load this page. Check your connection and refresh. |
| Logout | Sign out — confirm: “Sign out of the officer dashboard?” / Confirm: “Sign out” / Dismiss: “Stay signed in” |

No generic labels: never “Submit”, “OK”, “Click here”, “Save” as primary CTAs in this phase.

Destructive actions in Phase 1: logout confirmation only (data-safe). No report reject/resolve UI yet.

## Accessibility (Phase 1)

- WCAG 2.2 AA contrast for ink/muted on dominant and secondary
- Visible `:focus-visible` using accent ring
- Status and errors: text + color (never color alone)
- `prefers-reduced-motion`: instant state changes only (no entrance choreography)
- Locale control: accessible name “Language” / “Ngôn ngữ”
