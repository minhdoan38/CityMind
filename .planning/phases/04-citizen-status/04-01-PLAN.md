<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Untrusted citizen client → FastAPI POST /status | Token proof is the only credential; body may be forged or brute-forced. |
| FastAPI → Supabase service role | Hash lookup and report/history reads bypass RLS; must not over-return fields. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-01 | Information disclosure | POST /status verify failures | high | mitigate | Uniform 401 + identical detail for missing/expired/mismatch (CIT-03 / D-16); never 404 |
| T-04-02 | Elevation of privilege | Token↔report binding | high | mitigate | token_hash PK lookup then secrets.compare_digest on report_id (D-10) |
| T-04-03 | Information disclosure | CitizenStatusResponse | high | mitigate | Pydantic allowlist strips actor_id, recommendation, evidence, AI internals (CIT-02 / D-05) |
| T-04-04 | Spoofing / DoS | Status brute force | medium | mitigate | Separate status_limiter + XFF client_ip; 429 Retry-After (CIT-04 / D-17) |
| T-04-05 | Information disclosure | Logging | medium | mitigate | Never log plaintext token or echo token in error bodies (D-10) |
| T-04-SC | Tampering | Package installs | low | accept | No new pip packages in this plan |
</threat_model>
