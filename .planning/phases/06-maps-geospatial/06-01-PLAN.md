## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Officer client → /geo/pins | JWT required; same officer principal as list/export. |
| Geo pins response | Minimal pin projection only — no citizen narrative or tokens. |
| PostGIS RPC | Officer-scoped; no public anon access to pin endpoint. |
| docker-compose | Dev ergonomics; secrets via env files not committed. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-06-01 | Information Disclosure | /geo/pins payload | high | mitigate | D-20 field allowlist; tests assert forbidden keys absent. |
| T-06-02 | Spoofing | unauthenticated geo access | high | mitigate | require_officer on /geo/pins; 401 without JWT. |
| T-06-03 | Tampering | bbox injection | medium | mitigate | Validate west&lt;east, lat/lng ranges; parameterized RPC. |
| T-06-04 | Information Disclosure | unlocated_count leakage | low | accept | Aggregate count only; no report IDs for unlocated rows. |
| T-06-05 | Denial of Service | huge viewport query | medium | mitigate | Require finite bbox; consider row limit cap in RPC (e.g. 5000). |
</threat_model>
